基于io_uring的Java异步IO库
背景：在参与某个KV数据库训练时，发现Java标准库的DirectIO和AIO支持较差，且需要消耗额外的线程进行处理，同时发现未有Java的io_uring的实现，所以决定开发一个io_uring的Java异步IO库
  • 通过JEP454 引入的FFI API，实现pure Java的io_uring绑定，在参数填充，CQE收割的热路径上，无Native切换开销
  • 提供与posix api对位的异步IO API，从文件IO到网络IO,以及部分诸如EventFd，pipeFd的io_uring封装
  • 经过monoio的启发，引入所有权概念以及使用Thread-per-core的EventLoop设计，构建了无额外拷贝，且内存安全无竞态的异步IO API
  • 支持诸如Multi-shot，Provided buffers，IoUringBufferRing，Selected Buffer Read等特性，以及针对单线程io_uring的IORING_SETUP_SINGLE_ISSUER优化特性
  • 额外实现了基于APT/Reflection，以及字节码库等元编程工具，实现了声明式的FFI和FFM绑定，通过invokedyamic和methodhandle api实现了惰性绑定和接近于手写绑定的峰值性能
